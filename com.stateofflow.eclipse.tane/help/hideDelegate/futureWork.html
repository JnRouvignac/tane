Hide Delegate - Future Work

These may or may not get done as time and interest permits:

  • Suggest a method name in the wizard.
  • Guess parameter names in the wizard.
  • Allow editing of parameter names in the wizard.
  • Detect if an existing method's signature will clash with the new method's signature in the receiver.
  • Automatically apply the refactoring recursively.
  • Recognise when a parameter is an expression with the same receiver as the selected expression and avoid passing it as a parameter to the method created during refactoring. Instead, evaluate the expression inside the new method. e.g. currently, foo.getBar().doSomething(foo.getBaz()) will be refactored to foo.doSomething(foo.getBaz()). It should instead be refactored to foo.doSomething() and the implementation of Foo.doSomething() should look like getBar().doSomething(getBaz()).

If you are interested in helping with any of these (or with other enhancements or new refactorings), please get in touch via the website.
