Hide Delegate - Usage

Given the code at the start of the above example, select the code to be refactored in the Java editor:

Foo foo = new Foo();
…
foo.getBar().getBaz().doSomething();

Note that you can select any part of the sequence of invocations. You could, for example, have selected just getBar().getBaz() or getBaz().doSomething().

Now bring up the context sensitive menu and select the 'Hide Delegate...' menu item (note that this menu item is not under the Refactor sub-menu because we don't know how to get that to happen). The refactoring wizard that appears will check that appropriate preconditions have been met and then ask for some information:

  • A name for the new method.
  • The scope for replacement. This refactoring will search the selected scope for the same method invocation sequence that was selected and replace it with a call to the new method. This is a powerful way of replacing significant amounts of code with a better alternative, particularly if the code exhibits symptoms of 'procedural' thinking.

After entering the appropriate information, click 'Next'. The refactoring will work out what needs to be done and then show you a preview of the work to be done, as usual. If you are happy with the changes, click 'Finish' and the changes will be applied.
